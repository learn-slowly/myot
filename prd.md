# 내옷 myot — My Only Trend

> AI 기반 스타일 규칙 옷장 관리 앱 "유행은 따르지 않는다. 나만의 트렌드를 입는다."

---

## 1. 프로젝트 개요

### 1.1 한 줄 요약

자기 스타일 규칙이 있는 사람을 위한 AI 옷장 관리 + OOTD 착장 분석 앱

### 1.2 왜 만드는가

- 기존 옷장 앱들은 "일반 여성 패션" 범용 AI 추천에 초점
- 워크웨어/아이비/아메카지 같은 서브컬처 패션 스타일 규칙을 이해하는 앱이 없음
- 시즌별 소재 분리(울 니트가 여름에 추천되는 문제 등)가 제대로 안 됨
- "안 입는 옷 사지 않기" 같은 개인 원칙을 반영하는 앱이 없음

### 1.3 타겟 사용자

- 자기 스타일 축이 있는 사람 (워크웨어, 아이비, 미니멀, 아메카지 등)
- 옷을 많이 사기보다 가진 옷을 잘 입고 싶은 사람
- 지속가능한 소비에 관심 있는 사람
- "옷장에 옷은 많은데 입을 게 없다"고 느끼는 사람

### 1.4 경쟁 서비스

| 서비스 | 특징 | 약점 |
| --- | --- | --- |
| Acloset (에이클로젯) | 한국 앱, 400만 유저, AI 코디 추천 | 범용 추천, 서브컬처 미지원 |
| Whering | 영국, 랜덤 셔플 코디 | 시즌/소재 혼재 문제 |
| Indyx | 전문 아키비스트 서비스 | 비용 높음, 한국 미지원 |
| Pronti AI | OOTD + AI 채팅 | 영어 중심 |

### 1.5 차별화

| 기존 앱 | myot |
| --- | --- |
| 범용 AI 추천 | 스타일 축(워크웨어/아이비/힙합 등) 기반 규칙 |
| 시즌 필터 있지만 상의/아우터 혼재 | 시즌별 완전 분리 조합 (소재까지 고려) |
| 옷 사진 개별 등록 | OOTD 사진 → AI 자동 매칭 → 수동 보정 |
| 통계는 있지만 액션 없음 | 착용 데이터 → 처분/추가 추천까지 |
| 체형/퍼스널컬러 중심 | 개인 스타일 원칙 반영 |

## 2. 기능 명세

### 2.1 핵심 기능 (구현 완료)

#### 📸 OOTD (착장 기록)

- 사진 업로드 (카메라 촬영 + 갤러리 선택)
- Claude Vision API로 착장 분석 → 옷장 아이템 자동 매칭 (소품 포함)
- 사용자가 틀린 것 수정 가능 (삭제/추가 + 검색 기능)
- 저장 시 AI가 최종 아이템 목록 기반 코디 코멘트 재생성
- OOTD 사진을 Cloudinary에 업로드하여 기록에서 사진 확인 가능
- 착용 통계: 자주 입는 옷 TOP 5, 미착용 아이템 목록
- 최근 기록 타임라인 (사진 + AI 코멘트 + 아이템 태그)

#### ◫ 옷장

- 전체 아이템 카테고리별 보기 (Supabase DB 연동)
- 시즌별 필터 (봄/여름/가을/겨울)
- 아이템별 색상 도트 또는 사진 썸네일, 브랜드, 스타일 태그 표시
- **아이템 편집**: 이름, 카테고리, 브랜드, 색상, 시즌, 스타일 태그, 메모 수정
- **새 아이템 추가**: 추가 시 AI가 자동으로 기존 옷장 기반 코디 조합 생성 (최대 5개)
- **아이템 삭제** 가능
- **사진 등록**: Cloudinary CDN 업로드, 썸네일로 표시

#### ◈ 코디 (자유 선택 추천)

- 시즌 선택 → 하의/상의/아우터 중 **아무 아이템이나 먼저 선택 가능**
- 선택한 아이템 기반으로 나머지 슬롯 자동 필터링
- 추가 선택할수록 조합이 좁혀짐
- 선택한 슬롯은 해당 아이템만 결과에 표시 (다른 후보 제외)
- 조합 ★ 저장 가능
- 기분(편하게/깔끔하게/멋있게/공식자리) 태그

#### ☀ 오늘 뭐 입지 (날씨 + 무드 기반 추천)

- **Open-Meteo 날씨 연동**: 현재 기온/날씨 자동 가져옴 (위치 기반, 서울 기본값)
- 기온 → 시즌 자동 판단 (25°C+→여름, 15\~24→봄/가을, \~14→겨울)
- 비 오는 날: 비 관련 신발(캠퍼스 회색, 장화) 포함 조합 우선
- 최근 OOTD와 겹치지 않는 조합 우선 추천
- 기분 선택 + 시즌 수동 오버라이드 가능
- 조건에 맞는 조합 중 3개 추천 + "다른 조합 보기" 셔플

#### 🤔 살/말 (AI 구매 판단)

- 사고 싶은 옷 사진 업로드 → AI가 옷장 데이터 기반 분석
- 중복 체크: 비슷한 아이템이 이미 있는지
- 활용도: 기존 옷들과 새 조합 가능성
- 스타일 적합성: 워크웨어/아이비/힙합 스타일 축에 맞는지
- 색상 조화: 웜톤 어스톤 컬러 팔레트와의 조화
- 최종 판정: 🟢 살 / 🟡 고민 / 🔴 말
- "살/고민" 판정 시 찜 목록에 바로 추가 가능

#### ★ 찜 (위시리스트)

- **커스텀 상태 카테고리**: 기본 제공 + 사용자가 원하는 상태 자유 추가 (봄/가을에 찾기, 겨울에 찾기 등)
- 아이템별 상세 편집: 가격, 구매처 링크(URL), 상태, 메모, 사진
- 사진 업로드 (Cloudinary CDN), 목록에 썸네일 표시
- 구매 링크 클릭으로 바로 이동
- 저장한 코디 목록

### 2.2 온보딩 & 옷 등록 플로우

핵심 철학: **"입력하는 앱"이 아니라 "살면서 쌓이는 앱"** 기존 옷장 앱의 최대 이탈 원인 = "옷 등록하다 지쳐서 포기". myot은 OOTD가 등록 수단이자 핵심 기능.

#### 온보딩 (첫 실행)

1. **스타일 축 선택** — 워크웨어/아이비/미니멀/스트릿/아메카지 등에서 복수 선택 (탭 몇 번이면 끝)
2. **첫 OOTD 찍기** — "오늘 입은 옷부터 시작해볼까?" → 사진 촬영/업로드 → AI 분석 → 첫 아이템 등록
3. **끝** — 옷장 채우기를 강요하지 않음. 매일 OOTD 찍으면 자연스럽게 쌓임

#### 옷 등록 방법 (현실적인 것만)

| 방법 | 설명 | 시기 |
| --- | --- | --- |
| **OOTD 누적** | 매일 착장 찍으면 자연스럽게 옷장이 채워짐. 메인 등록 수단 | MVP |
| **옷 한 벌 촬영** | AI가 배경 제거 + 색상/카테고리/소재 자동 태깅 | MVP |
| **수동 추가** | 이름/색상/카테고리 직접 입력 (사진 없이도 가능) | MVP |
| **쇼핑몰 구매내역 연동** | 무신사/29CM/무탠다드 등에서 가져오기 | v2 |
| **택 사진 촬영** | 상품코드로 자동 등록 | v2 |

#### 옷장 성장 타임라인

```
1주차: OOTD 7번 → 자주 입는 옷 10~15벌 자동 등록
2~3주차: 대부분의 옷이 등록됨 (잘 안 입는 옷은 아직 미등록)
1개월: AI가 "이 옷들 보니까 워크웨어+아이비 스타일이네,
        이런 조합 규칙 만들어볼까?" 제안
        → 사용자는 "맞아/아니야"만 선택
2~3개월: 시즌 지나면서 미등록 옷도 자연스럽게 채워짐
1시즌 후: "이 옷 한 번도 안 입었어" → 처분 추천 시작
```

#### AI 스타일 규칙 자동 생성 (v2)

- 옷이 20벌 이상 쌓이면 AI가 스타일 패턴 분석
- "이 옷들 조합으로 이런 규칙 만들어봤어" 제안
- 사용자가 확인/수정 → 개인화된 코디 규칙 완성
- 개인 원칙도 학습 ("이 사람은 모자를 안 쓰네", "빼입기만 하네")

### 2.3 향후 기능 (v2 이후)

#### 착용 통계 고도화

- 월별/시즌별/연간 착용 리포트
- 아이템별 cost-per-wear 계산 (가격 입력 시)
- "한 시즌 안 입은 옷" 자동 알림 → 처분 추천
- 자주 입는 조합 패턴 분석 → 새 아이템 추천

#### 스타일 축 커스터마이징

- 온보딩: 스타일 축 선택 (워크웨어/아이비/미니멀/스트릿 등)
- 선택된 축 기반으로 AI가 코디 규칙 DB 적용
- 개인 원칙 등록 ("모자 안 씀", "빼입기 선호", "노란색은 어렵다" 등)

#### 다중 사용자 지원

- Supabase 연결 → 회원가입/로그인
- 사진 저장 (Supabase Storage)
- 착용 기록 DB 저장

#### 중고 연동

- 처분 추천된 옷 → 번개장터/당근 등 연결

## 3. 기술 스택

### 3.1 현재

- **프레임워크**: Next.js 15 + TypeScript
- **스타일링**: Tailwind CSS v4
- **폰트**: Pretendard Variable
- **DB**: Supabase PostgreSQL (ap-northeast-2 서울 리전)
- **이미지 저장**: Cloudinary CDN (서버사이드 signed upload)
- **AI 분석**: Anthropic API (Claude Sonnet) — Next.js API Route 경유 (키 보호)
  - `/api/analyze`: 이미지 + 텍스트 분석 (OOTD, 살/말)
  - `/api/analyze-text`: 텍스트 전용 분석 (코디 코멘트, 조합 생성)
  - `/api/upload`: Cloudinary 서버사이드 업로드
- **날씨**: Open-Meteo API (무료, 키 불필요, 상용 가능)
- **배포**: Vercel (GitHub 자동 배포, Preview + Production 환경)
- **PWA**: manifest.json + apple-touch-icon 설정 완료

### 3.2 향후 (v2)

- **인증**: Supabase Auth
- **알림**: 웹 푸시 또는 카카오 알림톡
- **스타일 축 온보딩**: 첫 실행 시 스타일 선택
- **착용 통계 고도화**: 월별/시즌별 리포트, cost-per-wear

## 5. 카테고리 & 시즌 규칙

### 5.1 카테고리 (12개)

하의, 긴팔 셔츠, 니트/가디건, 후디, 긴팔 티셔츠, 반팔 라운드티, 반팔 카라티, 아우터·겨울, 아우터·봄가을, 아우터·여름, 신발, 소품

### 5.2 시즌별 아이템 배정 규칙

**하의:**

- 여름 (3개): 그레이 카펜터, 중청 테이퍼드(얇은), 진청 스트레이트(얇은)
- 봄가을겨울 (4개): 그레이 카펜터, 샌드베이지 카펜터, 연청 스트레이트, 진청 스트레이트

**상의:**

- 여름: 반팔 라운드티 5개 + 반팔 카라티 3개 = 8개
- 봄가을: 긴팔셔츠 4 + 면니트 3 + 가디건 1 + 후디 5(크림,흙색,머스터드,블랙,버건디) + PEEPS 레이어드 + 럭비티 2 + 레이어드 라운드티 = 18개
- 겨울: 봄가을 전부 18개 + 크리켓 니트(울) + 케이블 니트 브라운 + 니트 후디 그레이 = 21개

**아우터 — 시즌별 완전 분리 (절대 섞이지 않음):**

- 겨울 (5): 롱패딩 블랙, 헤링본 블레이저, 오버핏 모직 블레이저, 플리스 점퍼, 롱코트
- 봄가을 (7): 네이비 블레이저, 바시티, 초어재킷, 필드재킷, 바람막이, 셔츠재킷, 트렌치코트
- 여름 (3): 짙은녹색 바람막이(매우 얇음), 블랙 블레이저, 베이지 블레이저

**신발:**

- 사계절: 뉴발 1080, 아디다스 캠퍼스, 머렐 랩트, 캠퍼 구두
- 봄가을겨울: + 노스페이스 등산화
- 여름: + 크록스

### 5.3 코디 조합 수

- 여름: 10개 조합
- 봄가을: 16개 조합
- 겨울: 12개 조합
- **총 38개 조합**
